2 Each subdirectory contains a Dockerfile that does something useful:
3 
4 * ``develop-ci`` is the Dockerfile used for running Cycamore on a continuous
5  integration service. This Dockerfile assumes that the current working
6  directory is a Cycamore repository - and that version of Cycamore is copied
7  into the Docker container and used for the build. The Dockerfile in the
8  Cycamore repository root is a symbolic link to this Dockerfile. This
9  Dockerfile uses the base image ``cyclus/cyclus:latest`` from the Docker hub
10  repository. This Docker container is uploaded as ``cyclus/Cycamore:latest`` on
11  the Docker hub when merging a PR on the `develop` branch of the Cycamore Github
12  repository.
13 
14 * ``master-ci`` is the Dockerfile used for running last stable Cycamore version
15  a continuous integration service. This Dockerfile works exactly as the
16  develop-ci one, except that it builds Cycamore against the
17  ``cyclus/cyclus:stable`` from the Docker hub repository. This Docker container
18  is also uploaded on the Docker hub as the ``cyclus/Cycamore:stable`` when
19  merging a PR on the `master` branch of Cycamore Github repository.
20 
21 * ``deb-ci`` contains a script which allows to build the Dockerfile require
22  to generate a Cycamore Debian package for the installation of Cycamore on
23  ubuntu. The script take the major version number of ubuntu as a parameter. It
24  builds Cycamore on the corresponding Ubuntu-LTS version, after installing all
25  Cycamore dependencies -including Cyclus- and generates the Cycamore Debian
26  installation package that need to be extracted from the Docker container
27  (tested on Ubuntu 14.04 & 16.04)
